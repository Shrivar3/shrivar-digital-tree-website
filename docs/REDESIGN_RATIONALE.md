# Redesign rationale

Two visual systems were generated before implementation.

- **Direction A — Editorial / human:** strong typographic restraint, warmer photography, calmer magazine rhythm.
- **Direction B — Spatial / data-led:** a living research tree organizes the page, evidence graphics behave like working instruments, and each section has a distinct spatial rhythm.

Scores use a 1–5 scale.

| Criterion | A | B |
| --- | ---: | ---: |
| Feels specifically like Shrivar | 4 | 5 |
| Investor / founder signal | 4 | 5 |
| Clarity | 5 | 4 |
| Originality | 3 | 5 |
| Project evidence | 3 | 5 |
| Visual quality | 4 | 5 |
| Implementation feasibility | 5 | 4 |
| Mobile resilience | 5 | 4 |
| Performance risk | 5 | 4 |
| **Total** | **38** | **41** |

Direction B was selected. It makes the tree useful information architecture and gives every project a visual explanation instead of a generic card. The implementation borrows Direction A’s serif restraint and human warmth to offset the more technical system.
